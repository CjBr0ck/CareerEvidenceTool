# Дизайн-система

Статус: **порожньо.**

Правила, що зв'язують токени й компоненти в цілісну систему: як з окремих елементів
збираються екрани.

Це не склад компонентів — склад лежить у [`../components/`](../components/).
Тут — принципи, за якими їх поєднують.

## Що має тут з'явитися

- Сітка й система простору.
- Типографічна шкала та ієрархія.
- Правила застосування кольору (де саме акцент, а де ні).
- Патерни: форми, списки, порожні стани, помилки, підтвердження.
- Доступність: контраст, фокус, робота з клавіатури. Для продукту, де ставка на
  швидке логування, клавіатурна навігація — не опція.
- Правила i18n: UA + EN, різна довжина рядків, ніяких склеєних із фрагментів фраз.

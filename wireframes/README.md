# Вайрфрейми

Статус: **порожньо.**

Структура екранів без візуального стилю: що де лежить, що головне, що другорядне.
Desktop-first — мобільні розкладки з'являються після того, як desktop усталився.

## Ключові екрани MVP

MVP = тільки збір і перегляд (див. бриф). Отже:

1. **Швидке додавання** — інбокс-режим. Головний екран продукту з погляду тертя:
   вставка посилання, заголовок, закрив. Усе інше має бути необов'язковим.
2. **Список записів** — перегляд, пошук, фільтр за тегами й датою.
3. **Картка запису** — читання й редагування, робота з доказами всіх чотирьох типів.
4. **Режим рев'ю** — розбір накопиченого в інбоксі: збагатити, структурувати, протегувати.

Насадки (самооцінка, CV-буллети, кейс) у MVP не проєктуються, але місце під точку
входу до них варто передбачити.

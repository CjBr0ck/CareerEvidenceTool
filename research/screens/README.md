# Скріншоти конкурентів

Знято 21 липня 2026, Playwright + Chromium, вікно 1440×900, подвійна щільність пікселів.
Розкладено за групами з аналізу: `hard/`, `soft/`, `aspirational/`.

Для кожної сторінки два файли:
- `*-hero.png` — перший екран, те, що людина бачить одразу;
- `*-full.png` — сторінка цілком.

## Позначка «доступ обмежений»

Екрани, за які нас не пустили, підписані **прямо на зображенні** — червона плашка
зверху з назвою екрана і причиною. Жовта плашка означає інше: акаунт створено,
але він чекає підтвердження з пошти.

Це зроблено навмисно на самому файлі, а не в описі: знімок стіни входу легко
переплутати з реальним екраном продукту, коли дивишся теку через півроку.

## Що вдалося побачити зсередини

- **eLabFTW** — має публічне демо без реєстрації (`demo.elabftw.net`, скидається щодоби).
  Пройдено: головна, список записів, картка запису, шаблони, форма створення.
  Файли `elabftw-01…08`. Саме тут знайдено найцінніший факт аналізу: у списку є запис
  `Untitled` — тобто зберегти без заголовка можна, обов'язкових полів нуль.
- **bragdoc.ai** — відкритий код, тому форми й схему бази читали напряму в репозиторії.
  Це надійніше за скріншоти.

## Куди не зайшли

| Продукт | Чому |
|---|---|
| BragBook | Тільки Google або пошта з підтвердженням |
| bragdoc.ai | Одноразове посилання на пошту, Google або GitHub |
| Bragduck | Акаунт створено, чекає підтвердження пошти |
| ImpactLogr | Акаунт створено, чекає підтвердження пошти |
| FourteenFish | Платна підписка |

Два акаунти зареєстровані на адреси з позначкою (`+bragduck`, `+impactlogr`),
щоб їх легко було відфільтрувати або видалити. Щоб дозняти справжні робочі екрани,
потрібно підтвердити пошту в цих двох листах.

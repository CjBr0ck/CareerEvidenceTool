# Концепція

Статус: **порожньо.**

Візуальний напрямок і характер продукту: як він виглядає й що відчуває користувач.

## Рамки з брифу

- **Аудиторія — фахівці з досвідом, не джуни.** Тон: інструмент для професіонала,
  не коуч. Без повчань, без гейміфікації, без пояснень «що таке досягнення».
- **Desktop-first**, робоче місце, ймовірно поруч із іншими робочими інструментами.
- **UA + EN** — довжина рядків різна, макети мають це витримувати.

## Що має тут з'явитися

- Мудборд і візуальний напрямок.
- 1–2 ключові екрани в кольорі як перевірка напрямку.
- Аргументація вибору — чому саме так, а не «красиво».

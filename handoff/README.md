# Хендофф

Статус: **порожньо.**

Те, що передається в розробку: специфікації, стани, крайні випадки, поведінка —
у формі, з якої можна писати код без здогадок.

## Що має тут з'явитися

- Специфікації екранів: розміри, відступи, поведінка при ресайзі.
- Крайні випадки: довгий заголовок, сотні записів, порожній стан, помилка збереження,
  файл завеликого розміру.
- Модель даних запису — узгоджена з брифом, включно з конфігурованою обов'язковістю полів.
- Формат експорту даних. Це пряме продовження обіцянки «дані твої» з брифу
  і одне з відкритих питань, які треба закрити тут.

## Умова

Хендофф має сенс лише після того, як усталилися вайрфрейми й компоненти.
Заповнювати його раніше — переписувати двічі.
